[LeetCode] Bulb Switcher/Linked List Cycle/Find the Duplicate Number

three math problems, two medium one hard.

Bulb Switcher

这个题其实没怎么说明白，示例给的也很不清楚，leetcode上不少题都是这样。。
其实就是一排n个灯泡，第一轮先打开所有的灯泡，第二轮toggle（就是on变off，off变on）从头到尾数是2的倍数的灯泡，第三轮toggle所有是3的倍数的灯泡，…，第n轮就只toggle第n个灯泡，要求第n轮后还剩多少个灯泡是亮着的。
如果灯泡m在第k轮改变状态，那么k一定是m的约数，那么灯泡最初是off，如果要在n轮后状态是on，m的约数一定要是奇数才可以，那么m一定是某个数的平方，然后只要计算小于n的平方数就可以了。代码只有一行，但想出来还是费了不少时间。

Linked List Cycle

判断一个链表里有没有环，这个很常见了，两个指针一个slow一个fast，slow每次走一步，fast每次走两步，如果有环，slow和fast最终会相遇重合，注意一些判定条件就可以了。

Find the Duplicate Number

一个大小是n+1的vector，里面装的是从1到n的整数。那根据鸽巢原理肯定有两个数是一样的，找出重复的那个数。
这道题比较难，首先要能想到把问题转化成带环链表，其次还要找出环的入口，需要一些证明工作。
为什么可以转换成带环链表举个栗子就明白了，比如

就可以形成一个0->1->3->2->4->1->3->…的带环链表，接下来同样是两个指针slow和fast，向前移动直到二者相遇于某一点。当slow和fast重合时，这时候再用一个指针finder从数组0开始，finder和slow每次都向前移动一步，当二者再次重合时，所在的点就是环的入口，也就是数组中重复的元素。接下来证明这一结论。

  1. 设从0到环的入口处距离是m，0到slow和fast重合点处的距离是L，环的长度是c，当slow和fast相遇时，slow走了L，fast走了2L，这样就一定有2L = L + k*c => L = k*c (k倍)；
  2. 设从环的入口处到slow和fast重合处的距离是a，有m + a = L；
  3. 那么就有m+a = k*c => m = (k-1)*c + c-a;
  4. 上面左边m就是finder从0到环入口走的距离，右边是slow从和fast重合点处走到环入口的距离；
  5. 从而就证明了finder和slow一定重合于环的入口。
